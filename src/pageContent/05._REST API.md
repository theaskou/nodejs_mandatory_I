# REST API conventions vs. Richardson Maturity Model

REST (Representational State Transfer): REST is a set of principles for designing API's. When an API follow the REST conventions, it is said to be RESTful. The most common format to transfer data in is JSON. It is not a strict requirement for REST APIs to use JSON as the data format, but it has become the most commonly used data format due to its simplicity and compatibility with web technologies.

The three REST conventions we follow:

1. Using the right HTTP verb

2. Ordering: GET, POST, PUT, PATCH, DELETE.

3. Naming (noun, plural, ) that maps to collections in your system.

Below is an example of a correctly designed REST API.

| Endpoint          | HTTP Verb | Description                          |
| ----------------- | --------- | ------------------------------------ |
| /exercises        | GET       | Retrieves all exercises              |
| /exercises/{id}   | GET       | Retrieves a single exercise          |
| /exercises        | POST      | Creates a single exercise resource   |
| /exercises/{id}   | PUT       | Updates a single exercise resource   |
| /exercises/{id}   | PATCH     | Updates part of an exercise resource |
| /exercises/{id}   | DELETE    | Deletes a single exercise resource   |
