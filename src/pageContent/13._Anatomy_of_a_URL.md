# Anatomy of a URL

We are looking at this URL:

https://www.google.com/search?hl=en&q=my%20search

1. **Protocol**: `https`
   - The protocol specifies the method used to transfer data over the web. Common protocols include `http` and `https`.

2. **Domain**: `google.com`
   - The domain name identifies the website. It consists of:
     - **Subdomain**: `www`
       - An optional part of the domain that can be used to organize different sections of a website.
     - **Top-Level Domain (TLD)**: `com`
       - The last part of the domain name, indicating the type of organization or country (e.g., `.com`, `.org`, `.net`).

3. **Endpoint**: `/search`
   - The endpoint specifies the resource or action on the server. It often represents a specific page or functionality within the website.

4. **Path Variable**: `/search/1`
   - Path variables are used in URLs to represent dynamic resources. For example, `/search/1` might represent a specific search result with ID `1`.

5. **Query String**: `?hl=en&q=my%20search`
   - The query string provides additional parameters to the URL. It starts with a `?` and includes key-value pairs separated by `&`.
     - **Query Parameter**: `hl=en` and `q=my%20search`
       - Query parameters are used to send data to the server. In this example, `hl=en` sets the language to English, and `q=my%20search` specifies the search query.

6. **Fragment**: `#Duration_and_tie-breaking_methods`
   - The fragment identifies a specific section within a webpage. It is indicated by a `#` and is often used for navigation within a page.
   - Example: `https://en.wikipedia.org/wiki/Association_football#Duration_and_tie-breaking_methods`
