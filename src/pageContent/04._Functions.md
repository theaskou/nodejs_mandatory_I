# Functions in JavaScript

In JavaScript, functions are refered to as "first-class citizens". This means that functions are treated as first-class entities, and they have the same status as other data types like numbers, strings, or objects.

The first-class citizen status of functions in JavaScript allows for powerful and flexible programming patterns, such as higher-order functions, closures, and functional programming techniques. **Functions as Values:** Functions can be assigned to variables, stored in data structures (like arrays or objects), and passed as arguments to other functions.

## Hoisting

Hoisting in JavaScript means that variable and function declarations are moved to the top of their containing scope during the compilation phase. This means that regardless of where variables and functions are declared, they are moved to the top of the scope, allowing them to be used before they are declared in the code.

This means, that JavaScript does not run sequencially.

Here is an example. The function is available, even though it is declared after it is called in this scope:

```
console.log(getRandomInt(0, 10));

function getRandomInt(min, max){
    return Math.floor(Math.random() * (max + 1 - min) - min);
}
```

Also notice, that there is no semicolon after functions in JavaScript. This is because hey are not statements that need termination. The JavaScript engine understands that the function declaration is complete when it encounters the closing brace.

## Function declarations

The example below is a traditional function declaration with the name sleepingAction
It takes a single parameter name and returns a string that includes the name in a sentence.
This function can be called by its name and passed as an argument to other functions.


The example below is a Function Expression: When functions are assigned to variables or used as expressions, they are part of a larger statement. Notice the semicolon, because this is a variable/statement, it terminates the entire statement, not just the function expression.

## Arrow functions

In the next example below, we see an anonymous function, because it doesn't have a name; instead, it's assigned to the variable getRandomIntArrowFunction.. More specifically it is an Arrow Function. The => syntax defines an arrow function.

```
const getRandomIntArrowFunction = () => {
    return Math.floor(Math.random() * (max + 1 - min) - min);
};
```

Arrow functions were introduced because of the need to binding the `this` keyword differently in a function – but this is beyond the scope of our curriculum in this class.

## Callback-functions

A callback-function is a function that is parsed as a parameter to another function, with the intention of possibly calling this function at a later time – which is the meaning behind the term "callback-function". \

```
function genericPerformer(name, action) {
    return action(name);
}

const codingAction = (name) => `${name} likes coding.`;

console.log(genericPerformer("Lasse", codingAction))

```

Notice that the `()` parathesies that normally comes after a function, are not included in the example below. This is done to avoid running the function immediately, and instead send a reference to the function. If you write `console.log(getRandomInt);`, the function-reference will be printed.

Below are some more examples of how to write functions.

### Using a Named Function as a Callback

```
function genericPerformer(name, action) {
    return action(name);
}

function sleepingAction(name) {
    return `${name} likes sleeping`;
}

console.log(genericPerformer("Andreas", sleepingAction))

```

### Using an arrow functions as a Callback

An arrow function is defined inline as the second argument to genericPerformer. This function takes a single parameter name and returns a string. Arrow functions provide a concise way to define functions, especially for short, single-expression functions.
Similar to the previous example, this arrow function is used as a callback within `genericPerformer`.

```
function genericPerformer(name, action) {
    return action(name);
}

console.log(genericPerformer("Tara", (name) => `${name} owns at boxing`));
```

### Arrow Function Assigned to a Variable

In the example below, an arrow function is assigned to the variable `boxingAction`. This function is equivalent to the inline arrow function in the previous example.

By assigning the arrow function to a variable, it can be reused multiple times in the code, just like a named function.

```
const boxingAction = (name) => `${name} owns at boxing`;
```
